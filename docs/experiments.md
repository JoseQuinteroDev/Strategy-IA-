# Experiments

## Trend Pullback V1 Gold Baseline

`baseline_trend_pullback_v1` is the current research focus. It replaces the previous BTC/ETH mean-reversion, Nasdaq ORB, and intraday hybrid baselines conceptually; those families remain available as legacy controls.

The initial market is XAUUSD/gold. The expected local file can be a 1-minute OHLCV CSV even if its filename says `5m`; the strategy derives M5 setup bars and M15 bias bars causally from M1.

### Strategy Layers

- M15 bias: long only when close > EMA45, EMA45 > SMA200, and EMA45 is rising versus 3 closed M15 bars ago; short is symmetric.
- M5 setup: RSI pullback/recovery, VWAP alignment, VWAP distance capped by ATR, and optional MACD confirmation.
- M1 trigger: for trigger variants, the M5 setup lives for 10 closed M1 bars; entry requires RSI cross plus a break of the prior 3 closed M1 candles.
- Risk: mandatory stop, 0.50% risk per trade, max 2 trades/day, max 2 consecutive losses/day, 1.5% daily loss limit, one open position, no averaging/martingale.
- Session gate: entries only during `09:00-11:00` and `14:00-16:30` in `Europe/Madrid`; positions are flattened at the end of each configured session window by default.
- Execution: next-bar open entries, conservative SL/TP collision handling, gap-to-open handling, fixed contract/lot sizing via `point_value`, spread filter, fee and slippage points.

### Variants

- `core_v1`: core RSI/VWAP pullback with M1 trigger and TP 2R.
- `core_v1_macd`: core plus M5 MACD line/signal and improving histogram.
- `core_v1_no_m1`: core setup without M1 trigger; enters from M5 setup confirmation.
- `core_v1_tp_1p5`: core trigger with TP 1.5R.
- `core_v1_macd_tp_1p5`: MACD confirmation with TP 1.5R.

### Ranking

The ranking is intentionally robustness-first, not curve-beauty-first. It rewards positive net PnL, expectancy, profit factor, controlled drawdown, daily loss containment, train/validation/test consistency, positive year/quarter ratios, and enough trades to avoid a fragile one-off result. Cost sensitivity x1.5/x2/x3 and an anchored walk-forward check are generated for the selected top candidate.

## Intraday Hybrid Contextual Baseline

`baseline_intraday_hybrid` is retained as the previous primary Nasdaq/NQ baseline and as a useful legacy control for comparison.

The old families remain in the repository for reproducibility and historical comparison, but they are legacy controls rather than the preferred PPO input baseline.

### Strategy Layers

- Macro / HTF context: price versus `EMA200 1H`, `EMA200 1H` slope, `ADX 1H`, and distance versus EMA.
- Intraday execution: concrete candidate trades generated by one of three modes.
- Mandatory time filter: new entries are allowed only during `14:00-19:00 UTC` by default. The strategy applies `outside_session: do not trade` before final entry validation, and the risk engine mirrors the same session gate.
- Risk: the existing `PropFirmRiskEngine` remains decoupled and enforces stop loss, daily loss, total drawdown, max trades per day, session gates, and max open positions.

The opening window is configurable in `configs/variants/baseline_intraday_hybrid.yaml` through `strategy.enforce_entry_session`, `strategy.entry_session_start_*`, `strategy.entry_session_end_*`, and `risk.session_*`. `strategy.close_on_session_end` controls whether an already-open trade is forcibly flattened at the configured session close or can continue until SL/TP/time stop.

### Variants

- `macro_pullback_continuation`: trades pullbacks into intraday value only when HTF bias agrees.
- `controlled_mean_reversion`: trades VWAP/value extensions only when the macro regime is not strongly trending.
- `compression_expansion_continuation`: trades expansion from compact intraday structure only with macro alignment.

### Ranking

The ranking is intentionally not frequency-first. It prioritizes:

- profitability: `net_pnl`, `expectancy`, `profit_factor`
- risk: `max_drawdown`, estimated daily loss, temporal stability
- activity: trades/year, trades/week, active weeks

This keeps the baseline suitable for future PPO work without teaching the agent low-quality overtrading.
